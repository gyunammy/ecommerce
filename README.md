# 캐싱 성능 개선 보고서

## 1. 개요

인기상품 조회 API에 Redis 캐싱을 적용하여 응답 속도를 개선하였습니다.

## 2. 테스트 환경

- **컨테이너**: TestContainers (MySQL 8.0, Redis 7.2-alpine)
- **테스트 데이터**: 15개 상품
- **조회 조건**: 조회수(VIEW_COUNT) 기준 Top 10
- **테스트 방법**: 동일 조건으로 3회 연속 조회하여 성능 측정

## 3. 적용 기술

- **Spring Cache Abstraction**: `@Cacheable` 애노테이션 활용
- **캐시 저장소**: Redis
- **캐시 키 전략**: `topProducts::{sortType}`
- **TTL(Time To Live)**: 26시간

## 4. 성능 측정 결과

| 호출 차수 | 캐싱 여부 | 응답 시간 | 비고 |
|---------|---------|----------|------|
| 1차 호출 | ❌ 캐싱 없음 | 84ms | DB 직접 조회 |
| 2차 호출 | ✅ 캐싱 있음 | 40ms | Redis 캐시 조회 |
| 3차 호출 | ✅ 캐싱 있음 | 5ms | Redis 캐시 조회 |

## 5. 결과 분석

### 5-1. 성능 개선 효과
- **첫 번째 캐시 적중**: 2.1배 개선 (84ms → 40ms)
- **안정화된 캐시 조회**: 16.8배 개선 (84ms → 5ms)
- **평균 응답 시간**: 약 22.5ms (캐시 적중 시)

### 5-2. 캐싱의 이점
1. **DB 부하 감소**: 반복적인 조회 쿼리가 Redis로 분산되어 DB 부담 완화
2. **응답 속도 향상**: 최대 16.8배 빠른 응답으로 사용자 경험 개선
3. **확장성**: Redis 클러스터 구성으로 수평 확장 가능

## 6. 결론

Redis 캐싱 적용으로 인기상품 조회 API의 응답 속도가 **최대 16.8배 향상**되었으며, DB 부하가 크게 감소하였습니다. 특히 동일한 데이터를 반복 조회하는 상황에서 캐시의 효과가 극대화되었습니다.
